# Job Search Criteria — EXAMPLE (fictional persona: Sam Chen, Senior PM)

Copy this to `criteria.md` and edit, or run `/setup` to generate it.
`/find-jobs` reads this file to build its LinkedIn searches and to decide which
postings to drop before they reach your inbox.

## Roles (job titles to search for)

One LinkedIn search runs per role, per location mode. Keep this list short —
three roles is plenty. Ten roles means twenty searches and a slow run.

- Senior Product Manager
- Lead Product Manager
- Principal Product Manager

## Locations

- Seattle metro (within ~30 miles of Seattle, WA)
- Remote (US-based only)
- Hybrid with a Seattle-area HQ

Postings that don't match one of these get dropped. "Remote — Canada only" and
"Remote — EU" are drops.

## Seniority

- **Keep:** Senior, Lead, Principal, Group, Staff
- **Drop:** VP and above, "Head of", Chief, Associate, Junior, Intern
- **Keep but flag:** Director-level — sometimes right, sometimes a stretch

## Compensation floor

$150,000 annual, or $75/hr for contract.

Postings that disclose comp below the floor get dropped. Postings that **don't
disclose** comp are kept — most don't, and dropping them would gut the inbox.

## Job types

- **Keep:** full-time, contract, contract-to-hire
- **Drop:** part-time, internship, temp-staffing-only

If the type is genuinely unclear, keep it.
